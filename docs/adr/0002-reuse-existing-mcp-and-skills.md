# ADR-0002 — 기존 MCP·스킬 재사용 우선, 얇은 자체 레이어

- **Status:** Accepted (2026-06-20) — **PRD v0.3 §5.4("자체 스킬셋부터 작성") 입장을 대체(supersede)**. 범위는 [ADR-0004](./0004-scope-claims-to-detect-and-recommend.md)로 보강(연합 → 감지·추천).
- **관련:** PRD v0.5 §4.3, §5.4, §6.6 · [ADR-0001](./0001-zero-friction-first.md) · [ADR-0004](./0004-scope-claims-to-detect-and-recommend.md)

## Context

각 에이전트(Claude/Codex/Gemini)에는 사용자가 이미 설정해 쓰는 **MCP 서버와 스킬**이 있다.
이를 Framein에서 다시 만들거나 다시 설정하게 하면 (a) 작업 중복, (b) 사용자 마찰 증가
([ADR-0001] 위반), (c) 유지보수·라이선스 부담이 생긴다.

핵심 통찰: **"이미 쓰던 MCP·스킬을 세 에이전트가 그대로 공유해서 쓸 수만 있어도 충분히
훌륭하다."** 즉 1차 가치는 "재구현"이 아니라 "연합(federate)"에 있다.

v0.3 PRD §5.4는 "기존 스킬을 코드째 재배포하지 않고 자체 작성한 Framein 고유 스킬셋부터
구축"하는 입장이었다. 방향을 바꾼다.

## Decision

1. **재사용 우선** — 각 에이전트의 **기존 MCP 설정과 스킬을 상속·인식**하여 프레임 전체에서
   쓸 수 있게 한다. 재구현하지 않는다(PRD 비목표에 추가). `frame skills`가 기존+자체를 함께
   카탈로그로 노출한다.
2. **얇은 자체 레이어** — 그 위에 **동조화에 꼭 필요한 최소한의 Framein 자체 MCP**
   (`read_adr/append_adr/read_memory/write_memory/get_role/acquire_lock/release_lock` 등)와
   **협업 특화 소수 스킬**(ADR 기록 흐름, 역할 위임, 교차 역할 호출)만 자체 작성한다.
3. **일반 워크플로(TDD·리서치 등)는 기존 스킬 재사용을 우선**하고, 없을 때만 최소 보강한다.
4. **재배포 안 함** — 타인의 스킬을 재배포하지 않는다(사용자 자신의 설정을 연합할 뿐).
   명백히 변형한 경우만 출처 표기.

## Consequences

- **연합 메커니즘**이 필요하다(PRD §6.6): 각 에이전트의 기존 MCP/스킬 설정을 읽어 균일하게
   노출. "어디까지 인식하고 어디부터 실행/프록시할지"의 약관·안전 경계는 열린 질문(PRD §11.4).
- **자체 구현 표면이 줄어** MVP가 가벼워지고 마찰이 준다([ADR-0001]과 정합).
- 차별점이 "스킬 카탈로그의 양"이 아니라 **"동조화 + 기존 자산 재사용"**으로 명확해진다.
- 라이선스/출처 부담이 작아진다(재배포 안 함).

## Alternatives considered

- **자체 스킬셋부터 폭넓게 구축(v0.3 입장)** — 기각. 작업량·마찰·유지보수 과다, 차별점도 약함.
- **모든 MCP를 Framein 프록시로 강제 경유** — 보류. 안전·약관·복잡도 리스크. 우선은 "인식·연합",
   프록시는 필요·안전성 확인 후.
